Pangolinfo.com 平台及产品详细问答 (Q&A)

一、平台定位与能力

Q1: Pangolin 的核心定位和目标是什么？
Pangolin (www.pangolinfo.com) 是一个全球数据洞察工具 [1]，其核心目标是驱动电商业务增长 [1]。它致力于为跨境电商与数据驱动型企业提供高质量的数据采集和智能分析能力 [1]。
Q2: Pangolin 的产品体系覆盖了哪些完整的业务流程？
Pangolin 的产品体系覆盖了从数据抓取、自动化管理到智能洞察的完整链路 [1]。这一完整的流程包括：数据采集、应用数据、提供洞察，以及执行建议 [1]。
Q3: Pangolin 在数据采集的规模和成功率方面表现如何？
Pangolin 具有强大的数据能力，日均数据请求处理量超过 **30M+** [1]，数据采集成功率高达 **99%** [1]。特别是在 Amazon 平台，Sponsored 广告位的采集率达到了 **98%** [2]。
Q4: Pangolin 的主要客户群体有哪些类型？
Pangolin 的客户遍布全球，主要涵盖四种类型的机构或企业：电商卖家与品牌商、广告与营销机构、数据与技术服务商，以及研究与咨询机构 [1]。

二、核心产品与接口 (API)

Q5: Pangolin 的三大产品组合分别是什么，它们针对的核心功能是什么？
Pangolin 的产品可单独或组合使用，包括：
  • Scrape API： 为技术团队准备的数据接口，提供稳定高效的结构化数据获取（采集） [2]。
  • Data Tracker (AMZ Data Tracker)： 业务团队也能用的自动化数据追踪工具（追踪），支持零代码操作和可视化配置 [2]。
  • DataScaler (增长扫描仪)： 全自动“增长扫描仪”（洞察），致力于将全网繁杂数据转化为驱动业务的可执行洞察 [2]。
Q6: Scrape API 在 Amazon 数据采集方面有哪些核心优势和能力？
Scrape API 提供了稳定、高效的数据接口服务，主要优势包括：
  • 采集全面性： 覆盖 Amazon 全站，包括商品详情、榜单、广告位、评论与关键词等全量数据 [3]。
  • 高时效性： 最快可实现分钟级数据更新 [2]。
  • 广告位精准度： 实现 98% 赞助广告位覆盖，并支持邮区采集 [2, 3]。
  • 高并发支持： 支持无限制的高并发调用 [3]。
Q7: Pangolin 提供了哪些针对 Google 搜索结果（SERP）和 AI 数据的接口？
Pangolin 提供了多种 SERP 相关的 API [4]：
  • **SERP API：** 提供关键词在搜索引擎中返回的搜索结果，辅助选品 [5]。该系列包括 SERP API、SERP Async API 和 SERP Batch API [4]。
  • **AI Overview SERP API：** **独家支持** 抓取、提取和解析部分搜索结果中包含的 **Google AI 概述模块** 内容，以帮助卖家理解用户真实意图 [4, 5]。
  • **Keyword Trends API：** 用于获取关键词在不同时间段和地区的热度变化趋势，支持跨区域对比和历史波动分析 [4, 5]。
Q8: AMZ Data Tracker 的使用流程是怎样的？
AMZ Data Tracker 的核心流程非常简单：**选关注对象 → 创建 Tracker → 看结果** [6]。用户可以图形化定义目标站点、字段与采集周期，并支持分钟、小时、日级的自动化执行 [3]。关注对象可以包括 ASIN、关键词、类目榜、新品榜、店铺或热卖榜等 [6]。

三、认证与计费规则

Q9: 用户如何开始使用 Pangolin 的 API，以及如何获取认证凭证？
用户需要先注册来获取 Token [7]。认证接口的 URL 是 POST https://scrapeapi.pangolinfo.com/api/v1/auth [7]。用户需提供注册邮箱和密码作为请求参数 [7]。该认证返回的 Token 是访问提交任务 API 的凭证，并且是长期有效的 [7]。
Q10: Pangolin 积点的有效期规则是什么？未用完的积点可以结转到下个月吗？
积点是按月为单位发放的，无论是月付还是年付 [8]。但积点仅在当月有效 [8]。到期未使用的积点将 **自动清零**，不可结转到下月使用，也不可透支 [8]。
Q11: Pangolin 的套餐有哪些付费方式和优惠？
Pangolin 提供两种主要的支付方式：**月度订阅**（每月自动续订，可随时关闭）和 **年度订阅** [8]。选择年度订阅的用户，一次性支付 12 个月的费用，可以享受 **8 折** 优惠 [8, 9]。
Q12: 不同的 API 调用和 Data Tracker 的积点消耗标准是怎样的？
不同的产品和接口有不同的积点消耗标准 [9]：
  • Amazon Scrape API：每次调用消耗 **1** 积点 [9]。
  • Amazon Scrape API (Raw)：每次调用消耗 **0.75** 积点 [9]。
  • AMZ Data Tracker：每页面消耗 **1.5** 积点 [9]。
  • SERP API (返回 10 条结果)：每次调用消耗 **0.5** 积点 [9]。
  • AI Overview SERP API：每次调用消耗 **2** 积点 [9]。
